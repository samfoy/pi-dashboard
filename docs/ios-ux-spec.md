## 1. Chat List / Sidebar UX

### ChatGPT

- **Layout**: Left-side drawer/sidebar, accessed via hamburger menu icon (top-left) or edge swipe from left. On iPad, the sidebar is persistent in landscape.
- **New Chat**: Prominent "New Chat" button at top of sidebar — uses a compose/pencil-in-square icon. Also accessible via a dedicated button in the top-right of the main chat view when sidebar is closed.
- **Conversation List**: Single flat list of conversations, sorted by most-recently-active first.
- **Temporal Grouping**: Conversations are grouped under section headers:
  - **Today**
  - **Yesterday**
  - **Previous 7 Days**
  - **Previous 30 Days**
  - **Month names** (e.g., "March", "February") for older conversations
  - Section headers are small, gray, uppercase or semi-bold labels.
- **Search**: Search bar at top of sidebar. Filters conversations by title in real-time as you type. Uses standard iOS search bar styling with a magnifying glass icon.
- **Swipe Actions**:
  - Swipe left on a conversation → **Delete** (red) and **Archive** (gray/blue) actions.
  - Swipe right → **Pin** conversation action (optional, depending on version).
- **Long-press Context Menu**: Long-press on a conversation reveals a context menu with: Rename, Share, Archive, Delete.
- **Conversation Titles**: Auto-generated by the model after the first exchange. Displayed as single-line truncated text. Users can rename via long-press → Rename.
- **Active Conversation Highlight**: Currently selected conversation has a subtle rounded-rect background highlight (light gray in light mode, slightly lighter dark gray in dark mode).
- **Sidebar Animation**: Slides in from left with a slight dimming overlay on the main content. Uses an interruptible spring animation. The main content shifts/scales slightly to the right.
- **Profile/Settings**: User avatar/profile icon at the bottom of the sidebar, tapping opens settings.

### Claude

- **Layout**: Full-screen conversation list as the main view (not a sidebar drawer). Uses a standard `UINavigationController` push pattern — tapping a conversation pushes to the chat view.
- **New Chat**: Floating action button or prominent button in the bottom-right area, plus a compose button in the navigation bar. The FAB uses a slight shadow and spring animation on tap.
- **Conversation List**: Flat list sorted by recency. Each row shows:
  - Conversation title (bold, single line, truncated)
  - Preview of last message (secondary gray text, 1-2 lines)
  - Relative timestamp on the right ("2h ago", "Yesterday")
- **Temporal Grouping**: Similar to ChatGPT — groups conversations under "Today", "Yesterday", "Last 7 Days", "Last 30 Days", then by month.
- **Search**: Standard search bar at top, tucked under the navigation bar (pull down to reveal). Filters by conversation title and content.
- **Swipe Actions**:
  - Swipe left → **Delete** (red destructive action).
  - No swipe-right action by default.
- **Long-press Context Menu**: Rename, Delete options.
- **Empty State**: When no conversations exist, shows a friendly illustration/icon and text like "Start a conversation" with a CTA button.

### Key Design Patterns to Adopt

| Feature | Recommendation |
|---|---|
| Grouping | Temporal sections (Today / Yesterday / Last 7 Days / Older by month) |
| Sort order | Most recently active first within each group |
| New chat button | Both in nav bar AND as a prominent quick-access point |
| Swipe actions | Left-swipe for destructive (delete) with confirmation; right-swipe for quick action (pin/archive) |
| Search | Real-time filtering, search bar at top of list |
| Conversation preview | Title + last message snippet + relative timestamp |

## 2. Message Bubble Design

### ChatGPT

- **User Messages**:
  - Right-aligned with a colored background bubble (dark teal/green in dark mode, light gray-green in light mode).
  - Rounded rectangle shape with larger corner radius (~18pt). All four corners rounded — NOT the "chat tail" style.
  - Text is white (dark mode) or dark (light mode).
  - Max width: ~80% of screen width. Short messages are narrower (bubble hugs the text).
  - No avatar displayed for user messages.
  - Leading padding to the left leaves roughly 20% gap from the left edge.

- **Assistant Messages**:
  - Left-aligned, full-width (edge-to-edge with padding, NO bubble background). ChatGPT uses a flat/card-less design for assistant messages — just text on the base background color.
  - Small circular avatar icon (OpenAI logo, green/black) at the top-left of the assistant's response, inline with the first line or slightly above.
  - The avatar is ~24pt diameter.
  - Text uses standard label color (primary text color for the theme).
  - Rich markdown content renders inline (no bubble container).

- **Spacing**:
  - ~12-16pt vertical spacing between message groups.
  - ~4-8pt spacing between consecutive messages from the same sender.
  - Horizontal padding: ~16pt from screen edges.

- **Timestamps**:
  - NOT always visible. Hidden by default to keep the UI clean.
  - Tapping or long-pressing a message can reveal the timestamp.
  - Some versions show timestamps as section separators between messages that are far apart in time.

### Claude

- **User Messages**:
  - Right-aligned with a distinct bubble background. Uses a warm beige/tan color in light mode, a muted dark brown/slate in dark mode.
  - Rounded rectangle, generous corner radius (~16-18pt).
  - Text is dark in light mode, light in dark mode.
  - Max width ~80% of screen.

- **Assistant Messages**:
  - Left-aligned. Claude uses a subtle, very light background card/bubble for assistant messages (light cream/off-white in light mode, slightly elevated dark surface in dark mode) — a softer approach than ChatGPT's no-bubble style.
  - Claude's logo/avatar (small circular, the sunburst icon) appears at the top-left.
  - Full-width layout for the content area.

- **Spacing**: Similar to ChatGPT — ~12-16pt between message pairs, tighter within a burst from the same sender.

- **Timestamps**: Minimal/hidden by default. May show relative time on interaction.

### Key Design Patterns to Adopt

| Feature | Recommendation |
|---|---|
| User messages | Right-aligned, colored bubble, rounded rect (~18pt radius), max 80% width |
| Assistant messages | Left-aligned, full-width, minimal or no bubble (flat on background), small avatar at top-left |
| Bubble shape | Continuous corner radius (no pointer/tail), using `UIBezierPath` with `roundedRect` or SwiftUI `.clipShape(RoundedRectangle(cornerRadius: 18, style: .continuous))` |
| Avatar | Small (24-28pt) circular icon for assistant only; no avatar for user |
| Timestamps | Hidden by default, revealed on tap or as section separators for time gaps >1hr |
| Max width | 75-85% of container width for user bubbles |

## 3. Streaming Text Animation

### ChatGPT

- **Text Appearance**: Text streams in **chunk by chunk** as tokens arrive from the API. Each chunk is typically 1-5 tokens (a few words). The text appends smoothly without jarring layout shifts.
- **Cursor/Caret**: A small blinking cursor (block cursor or thin line) appears at the end of the streaming text, giving a "typewriter" feel. The cursor pulses/blinks with ~0.5s interval.
- **Layout Animation**: As new text appends and the content grows, the message view expands smoothly. The scroll view adjusts with the content — no jumping. Uses implicit `UIView` animations or SwiftUI's built-in layout animation.
- **Fade Effect**: No per-character fade-in. Text appears instantly as each chunk arrives. The smoothness comes from the rapid token delivery and smooth scroll.
- **Word-by-Word Feel**: Because tokens align roughly with words/subwords, the visual effect is approximately word-by-word appearance.
- **Completion**: When streaming finishes, the cursor disappears (fades out), and action buttons (copy, regenerate, thumbs up/down) fade in below the message.

### Claude

- **Text Appearance**: Similar chunk-by-chunk streaming. Claude's streaming tends to arrive in slightly larger chunks.
- **Cursor**: Claude uses a more subtle approach — a gentle pulsing dot or no explicit cursor, just the text appearing.
- **Smooth Expansion**: The message container smoothly expands as content arrives. The layout animation is fluid — content below shifts down naturally.
- **Thinking Display**: Before the main response streams, Claude may show a "thinking" phase with a distinct visual treatment (see Section 8).

### Implementation Recommendations

```
Streaming UX Checklist:
├── Append text in chunks as received (do NOT buffer full response)
├── Use AttributedString / NSAttributedString for incremental markdown rendering
├── Animate layout changes with spring animation (duration: 0.25s, bounce: 0.0)
├── Show blinking cursor at end of text during streaming
│   ├── Small vertical bar or block cursor
│   ├── Blink interval: 0.5-0.6s
│   └── Fade out when streaming completes
├── Auto-scroll to bottom (see Section 6 for scroll behavior details)
├── Pre-render markdown incrementally (don't wait for full response)
└── On completion:
    ├── Fade out cursor (0.2s)
    └── Fade in action buttons (0.3s, slight delay)
```

### Key Polish Details

- **No layout jumps**: The most critical UX detail. When text wraps to a new line or a code block starts, the expansion must be smooth, not jarring.
- **Incremental markdown**: Render markdown as it arrives. Bold text starts rendering as soon as `**` is opened and closed. Code blocks render with syntax highlighting as they stream.
- **Performance**: Use `UITextView` or a custom text renderer that can efficiently append attributed text without re-rendering the entire message. Avoid re-parsing the full markdown string on every chunk.

## 4. Markdown Rendering

### ChatGPT

- **Bold / Italic**: Standard rendering. Bold is heavier weight of system font (`.semibold` or `.bold`). Italic uses the italic variant. Renders inline seamlessly.
- **Headers**: `#` through `###` render with increased font sizes and bold weight. Clear vertical spacing above/below (more than paragraph spacing). `H1` is ~24pt, `H2` ~20pt, `H3` ~18pt (approximate).
- **Lists**:
  - Unordered: Bullet character (•) with hanging indent. Nested lists are indented further.
  - Ordered: Numbered (1., 2., 3.) with hanging indent.
  - Proper line spacing within list items.
- **Code — Inline**: Monospace font (SF Mono or Menlo), with a subtle background highlight (slightly darker/lighter rounded rect behind the text). Corner radius ~4pt. Small horizontal padding (~4pt).
- **Code — Block**:
  - Full-width card/container with rounded corners (~12pt radius).
  - Distinct background color (dark gray in light mode, darker surface in dark mode).
  - **Header bar** at the top of the code block with:
    - **Language label** (left-aligned, small text, e.g., "python", "javascript").
    - **Copy button** (right-aligned, "Copy" text or clipboard icon).
  - Syntax highlighting with a standard dark theme (similar to One Dark or VS Code dark).
  - Horizontal scrolling for long lines (no wrapping inside code blocks).
  - Monospace font throughout (SF Mono, ~13-14pt).
  - When the copy button is tapped, it briefly changes to "Copied!" with a checkmark, then reverts after ~2s.
- **Tables**:
  - Rendered as formatted tables with borders/grid lines.
  - Alternating row backgrounds for readability.
  - Horizontal scrolling if the table exceeds screen width.
  - Header row is bold with a distinct background.
- **Links**: Rendered in accent color (teal/blue), tappable, opens in SFSafariViewController.
- **LaTeX/Math**: ChatGPT renders LaTeX math expressions inline and as blocks using a math renderer. Rendered as proper mathematical notation.
- **Blockquotes**: Left border bar (2-3pt wide, accent or gray color) with indented text and slightly muted text color.

### Claude

- **Similar rendering** to ChatGPT with minor stylistic differences:
  - Code blocks use a slightly different color scheme (Claude's warm color palette).
  - Code block header includes language label + copy button (same pattern).
  - Tables render similarly with clean grid lines.
  - Claude puts slightly more vertical spacing between elements.
- **Artifacts**: Claude has a special "artifact" rendering for longer code, documents, etc. — presented as a tappable card that expands to a full-screen view. This is unique to Claude.

### Code Block Design Specification

```
┌─────────────────────────────────────────┐
│ python                        📋 Copy   │  ← Header bar (height ~36pt)
├─────────────────────────────────────────┤     Background: darker surface
│                                         │
│  def hello_world():                     │  ← Code area
│      print("Hello, world!")             │     Font: SF Mono 13pt
│      return True                        │     Line height: 1.4
│                                         │     Padding: 16pt all sides
│                                         │     Syntax highlighting: on
└─────────────────────────────────────────┘
                                               Corner radius: 12pt
                                               Horizontal scroll if needed
```

### Markdown Rendering Priorities

| Priority | Element | Notes |
|---|---|---|
| P0 | Bold, italic, inline code | Core text formatting — must work |
| P0 | Code blocks with copy | Most-used feature in AI chat |
| P0 | Lists (ordered + unordered) | Very common in responses |
| P1 | Headers (H1-H3) | Structure for long responses |
| P1 | Links | Tappable, open in SFSafariViewController |
| P1 | Syntax highlighting | Significantly improves code readability |
| P2 | Tables | Less common but important |
| P2 | Blockquotes | Used for callouts/emphasis |
| P3 | LaTeX/Math | Niche but valuable for technical use |
| P3 | Horizontal rules | Simple decorative separator |

## 5. Keyboard & Input Handling

### ChatGPT

- **Input Area**: Fixed at bottom of screen, above the keyboard when active. Uses a `UITextField`/`UITextView` inside a rounded-rect container.
- **Growth Behavior**:
  - Starts as a single-line input (~44pt height).
  - Grows dynamically as the user types multiple lines.
  - Max height before scrolling: approximately 5-6 lines (~120pt), then the text view becomes scrollable internally.
  - Growth animation is smooth (spring animation, ~0.2s).
- **Send Button**:
  - Circular button with an upward arrow icon (↑).
  - Positioned at the right end of the input area, vertically centered (or bottom-aligned when input is multiline).
  - **Appears only when there is text** — hidden/disabled when input is empty. Uses a scale + fade animation to appear/disappear.
  - Active color: white arrow on black circle (light mode) or black arrow on white circle (dark mode).
  - Disabled state: grayed out or hidden entirely.
- **Keyboard Avoidance**: The entire input area + message list animates up with the keyboard. Uses `UIKit`'s keyboard notification animations matching the keyboard's own curve and duration. In SwiftUI, this is handled by `.scrollDismissesKeyboard(.interactively)` and keyboard-avoiding layout.
- **Dismiss Keyboard**: Scrolling the message list dismisses the keyboard interactively (drag-to-dismiss). Also dismissible by tapping outside the input area.

### Claude

- **Very similar** to ChatGPT's input behavior.
- **Input Container**: Rounded rect with a subtle border/outline. Slightly more visual separation from the background.
- **Growth**: Same dynamic growth pattern, max ~5 lines before internal scrolling.
- **Send Button**: Similar circular send button that appears when text is present.

### Key Implementation Details

```
Input Area Behavior:
├── Container: rounded rect, 12pt corner radius, 1pt border (light separator color)
├── Padding: 12pt horizontal, 8pt vertical inside container
├── Min height: 44pt (single line + padding)
├── Max height: ~120pt (5 lines) before internal scrolling
├── Send button:
│   ├── 32pt diameter circle
│   ├── Positioned: trailing edge, bottom-aligned (for multiline)
│   ├── Appears with scale animation (0 → 1, spring, 0.3s) when text is non-empty
│   └── Disabled: hidden or 0.3 opacity
├── Keyboard matching:
│   ├── Animate with keyboard (match curve + duration from notification)
│   ├── Interactive dismiss on scroll
│   └── Safe area: respect home indicator on Face ID devices
└── Return key: inserts newline (NOT send). Send is button-only.
```

## 6. Scroll Behavior

### ChatGPT

- **Auto-Scroll During Streaming**: Automatically scrolls to the bottom as new text arrives. Uses smooth, continuous scrolling (not jumping).
- **User Scroll Interruption**: If the user scrolls up during streaming, auto-scroll **pauses**. The app respects the user's intent to read previous content.
- **"Jump to Bottom" Button**: When the user has scrolled up and new content is below the viewport:
  - A floating circular button appears (bottom-center or bottom-right of the message list).
  - Shows a downward arrow (↓ chevron).
  - May include an unread count badge or a subtle pulsing animation to indicate new content.
  - Tapping it smoothly scrolls to the bottom and re-enables auto-scroll.
  - The button appears with a fade + slide-up animation and disappears with fade + slide-down.
- **Auto-Scroll Resume**: Auto-scroll resumes when:
  - User taps the "jump to bottom" button.
  - User manually scrolls back to the bottom.
- **Pull-to-Load**: No pull-to-refresh in chat view. Older messages are loaded by scrolling to the top (lazy loading if conversation is very long).
- **Scroll Indicator**: Standard iOS scroll indicator (thin bar on right edge). Flashes briefly on content change.

### Claude

- **Same core behavior** as ChatGPT.
- **Jump to Bottom**: Similar floating button with down arrow.
- **Scroll Deceleration**: Standard iOS deceleration rate (`.normal`). Both apps avoid `.fast` deceleration to keep scrolling feeling natural.

### Scroll Behavior State Machine

```
States:
  [AT_BOTTOM] ──user scrolls up──→ [SCROLLED_UP]
  [SCROLLED_UP] ──tap jump button──→ [AT_BOTTOM]
  [SCROLLED_UP] ──manual scroll to bottom──→ [AT_BOTTOM]
  
Behaviors by state:
  AT_BOTTOM:
    - Auto-scroll enabled (follow new content)
    - Jump-to-bottom button hidden
  
  SCROLLED_UP:
    - Auto-scroll disabled (new content arrives below, user stays in place)
    - Jump-to-bottom button visible
    - Optional: badge/indicator showing new content below
    
Detection:
  - "At bottom" = contentOffset.y >= contentSize.height - frame.height - threshold(~50pt)
  - Use scrollViewDidScroll delegate to track state
```

## 7. Haptics & Micro-Interactions

### ChatGPT

- **Send Message**: Light haptic feedback (`UIImpactFeedbackGenerator`, `.light` or `.medium`) when tapping the send button.
- **Copy Action**: Success haptic (`.success` notification feedback) when copying code or text. Paired with the "Copied!" visual confirmation.
- **Long-Press Context Menu**: Standard iOS context menu haptic (the system provides this automatically with `UIContextMenuInteraction`).
- **Button Presses**: Subtle haptic on most interactive elements (model selector, new chat, sidebar toggle).
- **Swipe Actions**: Haptic feedback at the swipe threshold (when action is "armed").
- **Spring Animations**: Used extensively:
  - Send button appear/disappear: spring with `response: 0.3, dampingFraction: 0.7`.
  - Sidebar open/close: spring with `response: 0.35, dampingFraction: 0.85`.
  - Button scale effects: buttons briefly scale down to ~0.95 on press, then spring back.
  - Model selector dropdown: spring animation for open/close.
- **Loading Dots**: The typing indicator (three dots) uses a staggered bounce animation — each dot bounces with a slight delay (0.15s offset between dots).

### Claude

- **Similar haptic patterns** — send, copy, long-press.
- **Response Complete**: Subtle haptic when a response finishes streaming.
- **Thinking Indicator**: Animated pulsing/breathing effect (opacity oscillation + slight scale) during the thinking phase.
- **Smooth Transitions**: Heavy use of `matchedGeometryEffect`-style animations for artifact cards expanding to full screen.

### Haptic Feedback Map

| Action | Haptic Type | Intensity |
|---|---|---|
| Send message | Impact, `.medium` | Standard |
| Copy text/code | Notification, `.success` | Standard |
| Long-press | System (automatic) | Standard |
| Swipe action threshold | Impact, `.light` | Standard |
| Button tap | Impact, `.light` | Standard |
| Error / failed send | Notification, `.error` | Standard |
| Pull-to-refresh | Impact, `.medium` | Standard |
| Toggle / switch | Impact, `.light` | Subtle |

### Animation Specifications

| Animation | Type | Duration | Parameters |
|---|---|---|---|
| Send button appear | Spring | 0.3s | damping: 0.7, response: 0.3 |
| Message appear | Spring | 0.25s | damping: 0.8, response: 0.25 |
| Jump-to-bottom appear | Ease + Fade | 0.2s | opacity 0→1, translateY 10→0 |
| Sidebar slide | Spring | 0.35s | damping: 0.85, response: 0.35 |
| Button press scale | Spring | 0.15s | scale 1.0→0.95→1.0 |
| Code "Copied!" label | Crossfade | 0.2s | opacity transition |
| Loading dots bounce | Repeat + Delay | 0.6s/cycle | 0.15s offset per dot |
| Cursor blink | Repeat | 1.0s/cycle | opacity 1→0→1 |

## 8. Tool Use / Thinking Indicators

### ChatGPT

- **Thinking/Reasoning** (o1/o3 models):
  - Shows "Thinking..." with an animated indicator (pulsing text or animated icon).
  - A collapsible section header like "Thought for X seconds" appears above the response.
  - Tapping it expands to show the thinking content (in a slightly muted/italic style).
  - The thinking text uses a distinct visual treatment — slightly smaller font, muted color, or contained in a collapsible card.
  - An animated timer or progress indication may show during thinking.

- **Web Search**:
  - Shows "Searching the web..." with an animated globe/search icon.
  - When complete, shows "Searched X sites" as a collapsible section.
  - Expanding shows the sources/citations as tappable links.

- **Code Interpreter / Analysis**:
  - Shows "Analyzing..." with a code/terminal icon animation.
  - Expandable section showing the code that was run and its output.
  - Progress indication for long-running code.

- **Image Generation (DALL-E)**:
  - Shows "Creating image..." with a generation progress animation.
  - Image appears with a fade-in when ready.
  - Tappable to view full-screen.

- **General Pattern**: All tool uses follow a consistent pattern:
  1. **Active state**: Animated label ("Searching...", "Thinking...") with a spinner or pulsing animation.
  2. **Completed state**: Collapsible summary ("Searched 5 sites", "Thought for 12 seconds").
  3. **Expandable detail**: Tap to see full details (sources, thinking text, code output).
  4. **Transition**: Smooth animation from active → completed state.

### Claude

- **Thinking** (Extended Thinking):
  - Shows a distinct "Thinking..." indicator with Claude's animated icon.
  - The thinking block is visually separated — often in a collapsible card with a slightly different background.
  - Shows duration: "Thought for X seconds."
  - Expandable to reveal the full thinking process.
  - The thinking text is displayed in a monospace or slightly different font to distinguish it from the final response.
  - A pulsing/breathing animation on the thinking indicator (gentle opacity oscillation 0.4-1.0, ~2s cycle).

- **Tool Use**:
  - Shows tool name and status ("Using calculator...", "Searching...").
  - Tool results displayed in a compact card format.
  - Collapsible after completion.

### Thinking/Tool Indicator Design

```
┌─────────────────────────────────────────┐
│ 💭 Thinking...                     ⌄    │  ← Active state (animated)
│   ████░░░░░░ (pulsing bar)              │     Pulsing opacity animation
└─────────────────────────────────────────┘

        ↓ completes → transitions to:

┌─────────────────────────────────────────┐
│ 💭 Thought for 8 seconds           ⌄   │  ← Collapsed state (tappable)
└─────────────────────────────────────────┘

        ↓ user taps → expands:

┌─────────────────────────────────────────┐
│ 💭 Thought for 8 seconds           ⌃   │
├─────────────────────────────────────────┤
│ Let me analyze this step by step...     │  ← Thinking content
│ First, I need to consider...            │     Muted text color
│ The key factors are...                  │     Slightly smaller font
│ ...                                     │     Max height with scroll
└─────────────────────────────────────────┘
```

## 9. Error States & Loading

### ChatGPT

- **Connection Lost**:
  - A banner slides down from the top (below the nav bar) or appears inline: "You're offline" or "Connection lost."
  - Uses a yellow/amber warning color or red for errors.
  - Includes a "Retry" or "Try Again" button.
  - Banner dismisses automatically when connection is restored.

- **Message Send Failure**:
  - The message bubble shows a red exclamation mark (⚠️) icon.
  - Tapping the error icon shows options: "Retry" or "Delete."
  - The message remains in the conversation with a visual error state (slightly faded or with error indicator).

- **Streaming Interruption**:
  - If streaming is interrupted mid-response, the partial text remains visible.
  - A "Continue generating" button appears below the partial response.
  - Or a "Regenerate" button to retry from scratch.

- **Rate Limiting**:
  - Shows an inline message about rate limits with a timer: "You've reached the limit. Try again in X minutes."
  - Clean, non-alarming design.

- **Empty States**:
  - New conversation: Shows a centered welcome message with suggested prompts/starters.
  - ChatGPT shows "How can I help you today?" with 4 suggested prompt cards in a 2x2 grid.
  - Suggested prompts are tappable cards with icons and short descriptions.

- **Loading**:
  - Initial app load: Splash screen → main UI (no skeleton loaders).
  - Conversation load: Messages appear quickly (likely cached locally). No skeleton loaders for individual messages.
  - The typing indicator (three bouncing dots) serves as the loading state for responses.

### Claude

- **Similar error patterns** to ChatGPT.
- **Empty State**: Claude shows a more branded empty state with its logo and a greeting message. Suggested conversation starters may appear as chips/pills.
- **Error Messages**: Tend to be slightly more descriptive/friendly in tone.
- **Offline Banner**: Similar slide-down banner pattern.

### Error State Specifications

| State | Visual Treatment | Action |
|---|---|---|
| Offline | Top banner, amber/yellow, network icon | Auto-dismiss on reconnect |
| Send failed | Red (!) icon on message bubble | Tap → Retry / Delete |
| Stream interrupted | Partial text shown | "Continue generating" button |
| Rate limited | Inline card with timer | Wait or upgrade CTA |
| Server error (500) | Inline error message | "Try Again" button |
| Empty conversation | Centered welcome + suggestions | Tap suggestion or type |
| Loading response | Three bouncing dots indicator | Cancel button (stop generating) |
| Conversation loading | Brief spinner or instant (cached) | None needed |

## 10. Navigation Patterns

### ChatGPT

- **Primary Navigation**: Sidebar drawer (hamburger menu) pattern.
  - No bottom tab bar.
  - Sidebar contains: conversation list, search, new chat, profile/settings.
  - Main area is the chat view.
- **Navigation Stack**: Flat — sidebar → chat is not a push navigation; the sidebar overlays.
- **Model Selector**: Dropdown/picker at the top of the chat view (centered in nav bar area). Tapping opens a picker/popover to switch models.
- **Settings**: Sheet presentation (modal) from sidebar → profile icon.
- **Sheet Presentations**: Used for settings, sharing, image preview. Standard iOS `.sheet` or `.fullScreenCover` with drag-to-dismiss.
- **iPad Adaptation**: Split view — sidebar is persistent on the left, chat view on the right. Standard `UISplitViewController` behavior.

### Claude

- **Primary Navigation**: Tab bar at bottom with:
  - **Chat** (conversation list as home)
  - **Recents** or integrated into main list
  - **Settings/Profile**
- **Navigation Stack**: Standard push navigation — conversation list → chat detail (push). Back button to return to list.
- **Model Selector**: Available in the chat view or as part of conversation settings. May be in the input area or nav bar.
- **Sheet Presentations**: Settings, account management as modal sheets.

### Recommended Navigation Architecture

```
For a new app, combine the best patterns:

Option A — Sidebar (ChatGPT-style):
├── Sidebar (leading)
│   ├── New Chat button
│   ├── Search bar
│   ├── Conversation list (grouped by time)
│   └── Settings / Profile at bottom
└── Main Content (chat view)
    ├── Nav bar: model selector (center), sidebar toggle (leading)
    ├── Message list (scrollable)
    └── Input area (bottom)

Option B — Tab + NavigationStack (Claude-style):
├── Tab: Chats
│   ├── NavigationStack
│   │   ├── Conversation list (root)
│   │   └── Chat detail (pushed)
│   └── New chat button (nav bar or FAB)
├── Tab: Explore / Discover
└── Tab: Settings

Recommendation: Option A (sidebar) for AI chat apps — it keeps the focus on the 
conversation while providing quick access to history. The single-purpose nature 
of a chat app doesn't benefit from tab bar navigation.
```

## 11. Dark Mode / Theming

### ChatGPT

- **Light Mode**:
  - Background: Pure white (#FFFFFF) or very light gray (#F7F7F8).
  - User bubbles: Light gray (#E9E9EB) or soft teal.
  - Assistant text: On white/light gray background (no bubble).
  - Code blocks: Dark background (#1E1E1E or #2D2D2D) even in light mode.
  - Accent color: Teal/green (#10A37F, the OpenAI green).
  - Navigation: White with subtle bottom border.

- **Dark Mode**:
  - Background: Very dark gray (#212121 or #343541 — the signature ChatGPT dark color).
  - User bubbles: Slightly lighter dark surface (#2F2F2F) or dark teal.
  - Assistant text: On the base dark background.
  - Code blocks: Slightly darker than the background (#1A1A1A).
  - Accent color: Same teal/green, slightly brighter for contrast.
  - Navigation: Dark surface with subtle separator.
  - **Notable**: ChatGPT's dark mode is the iconic/default experience. Many users associate the dark gray background with ChatGPT.

- **Adaptation**: Uses `@Environment(\.colorScheme)` / `UITraitCollection` to automatically adapt. No manual toggle in most versions (follows system setting), though some versions offer light/dark/system toggle.

### Claude

- **Light Mode**:
  - Background: Warm off-white/cream (#FAF9F6 or similar).
  - User bubbles: Warm beige/light brown (#EAE4D9 or similar).
  - Assistant messages: On slightly warm white surface.
  - Accent color: Warm orange/terracotta (Claude's brand color, #D97757 or similar).
  - Code blocks: Dark background, similar to ChatGPT.
  - Overall feel: Warmer, more literary/approachable than ChatGPT.

- **Dark Mode**:
  - Background: Warm dark (#2A2520 or similar warm dark).
  - Maintains the warm color temperature (not blue-gray like many dark modes).
  - Accent color: Adjusted for dark mode contrast.
  - User bubbles: Warm dark brown.
  - Text: Warm off-white (not pure white).

- **Key Difference**: Claude uses warm tones throughout (beige, cream, terracotta) while ChatGPT uses cooler tones (teal, gray). Claude's design feels more like reading a book; ChatGPT feels more like a technical tool.

## 12. Input Area Details

### ChatGPT

- **Placeholder Text**: "Message ChatGPT..." or "Ask anything" in muted gray text. Disappears when typing begins.
- **Attachment Options**:
  - **Plus (+) button** to the left of the input field, opens an action sheet/menu with:
    - 📷 Camera (take a photo)
    - 🖼 Photo Library (pick from photos)
    - 📁 Files (document picker)
  - Attachments show as thumbnail previews above the input field with an (×) remove button.
- **Voice Input**:
  - Microphone button appears when the text field is empty (replaces send button position, or adjacent).
  - Tapping starts voice recording with a visual waveform indicator.
  - Voice mode (advanced): Full-screen voice conversation UI with animated orb/visualization.
- **Send Button Behavior**:
  - Hidden or disabled when text field is empty.
  - Appears with a spring scale animation when text is entered.
  - During streaming: The send button transforms into a **stop button** (square icon in circle) to cancel generation. This transformation uses a smooth morph animation.
- **Model Selector**: Above or near the input area (or in the nav bar). Shows current model name as a tappable pill/chip.
- **Input Area Background**: Slightly elevated from the main background. Subtle shadow or border to separate from the message list. Rounded rectangle container.
- **Safe Area**: Properly respects bottom safe area on devices with home indicator. Input area sits above the home indicator with appropriate padding.

### Claude

- **Placeholder Text**: "Message Claude..." or similar.
- **Attachments**:
  - Similar plus (+) button for attachments.
  - Supports images and files.
  - Thumbnail preview for attached images.
- **Voice Input**: Microphone button for voice transcription (not a full voice mode like ChatGPT).
- **Send Button**: Same pattern — appears when text is present, morphs to stop during generation.
- **Context Window Indicator**: Claude may show a subtle indicator of context usage or conversation length.

### Input Area Layout Specification

```
┌─────────────────────────────────────────────────┐
│  [Image Thumbnails Row - if attachments]    [×]  │  ← Attachment preview area
├─────────────────────────────────────────────────┤     (only shown when attachments exist)
│                                                  │
│  [+]  Message assistant...              [🎤/➤]  │  ← Main input row
│                                                  │
└─────────────────────────────────────────────────┘

Layout details:
├── Container: rounded rect, 12-16pt corner radius
├── Background: elevated surface color with 1pt border or subtle shadow
├── Plus (+) button: leading, 32pt, opens attachment picker
├── Text field: flexible width, center
│   ├── Placeholder: muted secondary text color
│   ├── Font: system 16pt (body)
│   ├── Min lines: 1
│   ├── Max lines: 5 (then scrolls internally)
│   └── Padding: 8pt vertical, 12pt horizontal
├── Send button (when text present): trailing, 32pt diameter
│   ├── Icon: arrow.up in circle
│   ├── Appear animation: scale spring (0→1)
│   └── During streaming: morphs to stop.circle
├── Mic button (when text empty): trailing, replaces send button
│   └── Icon: mic.fill
├── Bottom padding: safe area bottom + 8pt
└── Horizontal padding: 16pt from screen edges
```

### Send Button State Machine

```
States:
  IDLE (no text):
    - Send button: hidden (or show mic button)
    - Plus button: visible
    
  COMPOSING (has text):
    - Send button: visible (arrow up icon, active color)
    - Plus button: visible
    - Tapping send → sends message, clears text, transitions to STREAMING
    
  STREAMING (response in progress):
    - Send button: transforms to STOP button (square icon)
    - Text field: may be disabled or allow queuing
    - Tapping stop → cancels generation, transitions to IDLE
    
  ERROR (send failed):
    - Show retry indicator on failed message
    - Input returns to COMPOSING state with original text
```

## 13. Accessibility

### ChatGPT

- **VoiceOver**:
  - All interactive elements have accessibility labels and hints.
  - Messages are announced as: "You said: [message text]" and "[Assistant name] said: [message text]."
  - Code blocks have "Copy code" as an accessible action.
  - The sidebar is announced as a navigation region.
  - Send button: "Send message" label, "Double tap to send" hint.
  - Streaming progress is announced periodically (not every token).

- **Dynamic Type**:
  - Supports Dynamic Type — text scales with system text size setting.
  - Uses system text styles (`.body`, `.headline`, `.caption`) or custom styles that respect `UIContentSizeCategory`.
  - Code blocks may cap their font scaling to prevent overflow (or enable horizontal scrolling).
  - Bubble widths and layout adapt to larger text sizes.

- **Reduced Motion**:
  - Respects `UIAccessibility.isReduceMotionEnabled` / `@Environment(\.accessibilityReduceMotion)`.
  - When enabled: spring animations become simple fades or instant transitions.
  - Streaming cursor: still visible but may not blink (or reduced blink).
  - Loading dots: simplified animation or static "..." text.

- **Color Contrast**: Meets WCAG AA contrast ratios (4.5:1 for body text, 3:1 for large text).
- **Bold Text**: Responds to system bold text preference.
- **Switch Control / Full Keyboard Access**: Interactive elements are reachable.

### Claude

- **Similar accessibility support** — VoiceOver labels, Dynamic Type, Reduced Motion.
- **Thinking blocks**: Announced as "Claude is thinking" during the thinking phase, then the result is announced.

### Accessibility Implementation Checklist

```
Required:
├── VoiceOver
│   ├── All buttons: accessibilityLabel + accessibilityHint
│   ├── Messages: role = .text, label includes sender + content
│   ├── Images: accessibilityLabel with description
│   ├── Code blocks: "Code block, [language]. [content]. Double tap to copy."
│   ├── Streaming: announce "Response in progress" then final content
│   └── Navigation: proper traits (.header, .button, .link, etc.)
├── Dynamic Type
│   ├── Use .font(.body) / .font(.headline) etc. (system styles)
│   ├── Or: UIFontMetrics for custom fonts
│   ├── Test at all size categories (xSmall → AX5)
│   └── Ensure layout doesn't break at largest sizes
├── Reduced Motion
│   ├── Check AccessibilitySettings.isReduceMotionEnabled
│   ├── Replace springs with linear/easeInOut (0.2s)
│   ├── Disable cursor blink, loading dot bounce
│   └── Keep functional animations (scroll to bottom) but simplify
├── Color / Contrast
│   ├── All text: minimum 4.5:1 contrast ratio
│   ├── Interactive elements: minimum 3:1
│   └── Don't rely on color alone for status (add icons/text)
├── Bold Text
│   ├── Respond to legibilityWeight environment value
│   └── Increase font weight when bold text is enabled
└── Keyboard / Switch Control
    ├── Tab order is logical
    └── All actions reachable without touch
```

## Appendix B: SwiftUI Component Sketch

```swift
// High-level component architecture for reference

struct ChatView: View {
    var body: some View {
        VStack(spacing: 0) {
            // Navigation bar
            ChatNavigationBar(model: selectedModel)
            
            // Message list
            ScrollViewReader { proxy in
                ScrollView {
                    LazyVStack(spacing: 12) {
                        ForEach(messages) { message in
                            MessageBubbleView(message: message)
                                .id(message.id)
                        }
                        
                        if isStreaming {
                            StreamingIndicator()
                        }
                    }
                    .padding(.horizontal, 16)
                }
                .scrollDismissesKeyboard(.interactively)
                .onChange(of: messages.count) { scrollToBottom(proxy) }
            }
            .overlay(alignment: .bottom) {
                if showJumpToBottom {
                    JumpToBottomButton { scrollToBottom() }
                        .transition(.move(edge: .bottom).combined(with: .opacity))
                }
            }
            
            // Input area
            ChatInputArea(
                text: $inputText,
                onSend: sendMessage,
                onStop: stopStreaming,
                isStreaming: isStreaming,
                onAttach: showAttachmentPicker
            )
        }
    }
}

struct MessageBubbleView: View {
    let message: ChatMessage
    
    var body: some View {
        HStack(alignment: .top, spacing: 8) {
            if message.role == .assistant {
                AssistantAvatar()
            }
            
            if message.role == .user {
                Spacer(minLength: UIScreen.main.bounds.width * 0.2)
            }
            
            MarkdownContentView(content: message.content)
                .padding(12)
                .background(message.role == .user ? Color.userBubble : Color.clear)
                .clipShape(RoundedRectangle(cornerRadius: 18, style: .continuous))
            
            if message.role == .assistant {
                Spacer(minLength: 0)
            }
        }
    }
}
```
